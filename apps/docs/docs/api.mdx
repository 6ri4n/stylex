# API Reference

StyleX aims to have a minimal API surface. There are only a few core functions
to learn.

## `stylex.create`

```ts
function create<T extends {[key: string]: RawStyles | (...any[]) => RawStyles}>(
  styles: T,
): {[Key in keyof T]: StyleXStyles<T[Key]>};
```

Takes a map of style objects and returns an object of compiled StyleX styles.
These StyleX styles is mapping of keys to class names.

## `stylex.spread`

```ts
function spread(styles: StyleXStyles | StyleXStyles[]): {
  className: string;
  style: {[key: string]: string};
};
```

Takes a list of StyleX styles and returns an object with two keys: `className`
and `style`.

These can be spread onto an element to apply the styles directly or the keys can
be destructured and used to apply the styles to an element manually.

## `stylex()` _(legacy)_

```ts
function stylex(...styles: Array<StaticStyles>): string;
```

Similar to `stylex.spread` it takes one or more StyleX styles and returns a
class string.

**NOTE**: This function does not support dynamic styles.

## `stylex.createVars`

```ts
function createVars<Styles extends {[key: string]: Value}>(
  styles: Styles,
): Vars<{[key in keyof Styles]: string}>;
```

Create a set of global CSS Variables or Custom Properties that can be imported
and used within `stylex.create` calls anywhere within your codebase.

Analagous to `React.createContext`.

## `stylex.varsProvider`

```ts
function varsProvider(
  vars: Vars,
  overrides: {
    [key: keyof Vars]: string;
  },
): StyleXStyles;
```

Takes a variable set created with `createVars` and an object of overrides. It
returns a `StyleXStyles` object containing a className to apply a theme with the
overrides you define.

## Utilities

### `stylex.firstThatWorks`

```ts
function firstThatWorks<Values extends Array<string | number>>(
  ...styles: Values
): Values[number];
```

This is a simple utility function that let's you declare a list of fallback
values for a style property.

All of them are applied in the generated styles such that the first supported
style within the list takes effect within the browser.

### `stylex.keyframes`

```ts
function keyframes(frames: {[key: string]: RawStyles}): string;
```

`stylex.keyframes` takes a list of keyframes, creates a `@keyframes` rule and
returns the name.

### `stylex.include`

```ts
function include(styles: StaticStyleObj): RawStylesObj;
```

This function is similar to `@include` in Sass. It lets you take the result of a
`stylex.create`, usually from a different file and include it within a
`stylex.create` call locally.

```ts
import stylex from '@stylexjs/stylex';
import {otherStyles} from './otherStyles';

const styles = stylex.create({
  foo: {
    ...stylex.include(otherStyles.bar),
    color: 'red',
    marginTop: 0,
  },
});
```

**NOTE**: For the best results, always spread the result of `stylex.include`
before adding other local styles.

**NOTE**: You can only spread the result of `stylex.include` at the top level of
your styles.
